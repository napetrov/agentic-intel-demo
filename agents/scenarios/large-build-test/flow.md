# Guided Scenario Flow — Large Build/Test

## Demo intent

Show a larger System A engineering workload with explicit scale-up framing, visible build/test steps, and concrete completion evidence.

## Required opening

Start with:
`Starting large build/test demo`

Then briefly state the flow:
1. inspect target repo/task
2. select large execution profile
3. run environment checks
4. execute build/test sequence
5. collect logs and summarize outcome

## Scenario contract

- route: `local_large`
- use System A large-profile framing
- do not offload to System B unless the policy explicitly changes

## Minimum evidence to show

Include:
- profile selection statement
- build/test command evidence
- success/failure checkpoint
- final summary with next action
